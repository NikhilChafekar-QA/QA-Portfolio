1. Objective
To validate the end-to-end functionality of the e-commerce application, covering all major modules including Login, Search, Product Details, Cart, Checkout, and Payment.

2. Scope
In Scope
Functional Testing
UI/UX Testing
Regression Testing
Smoke Testing
Sanity Testing

Out of Scope
Load/Performance Testing
Security/Penetration Testing

3. Test Approach
Manual testing will be executed based on the Software Testing Life Cycle (STLC).
Test cases will be written and executed per sprint.
Defects will be logged and tracked in Jira.
Test cycles include smoke, functional, and regression phases.

4. Entry Criteria
All functional and UI requirements are approved.
Test environment is fully configured and stable.
Required test data is prepared and accessible.

5. Exit Criteria
All critical and high-priority test cases are executed.
No High or Critical severity defects remain open.
Test Summary Report is completed and reviewed.

6. Deliverables
Test Cases
Bug/Defect Reports
Test Summary Report
Requirements Traceability Matrix (RTM)
